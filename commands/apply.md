# /apply — Execute Bounded Automation Plan

`/apply` executes the execution plan generated by `/review-pr`. The default posture is still safe: dry-run unless automation is enabled and `dryRunByDefault` is disabled.

## Prerequisites

Before running `/apply`, you MUST complete ALL of the following:

1. **Run `/review-pr`** and inspect the generated execution plan
2. **Run `/verify`** on the current branch state — it must return PASS
3. **Confirm policy status** — no blocked policy decisions may remain
4. **Confirm target branch safety** — `/apply` never targets a protected branch directly

If any prerequisite is not met, resolve it before running `/apply`.

## Execution

Enable automation in your omni-link config:

```json
{
  "simulateOnly": false,
  "automation": {
    "enabled": true,
    "dryRunByDefault": false
  }
}
```

Then run `/apply`. omni-link will use the saved execution plan, create bounded branches, and optionally create a PR if GitHub automation is enabled.

## What Gets Executed

When you run `/apply`:

1. Loads the saved review artifact
2. Reads the generated execution plan
3. Evaluates policy and risk gates
4. Creates generated branches in affected repos
5. Optionally creates a PR when GitHub automation is enabled

## Safety Notes

- `/apply` never writes directly to `main` or any configured protected branch
- `/apply` respects policy thresholds and aborts on blocked decisions
- In dry-run mode, `/apply` returns the plan without creating branches

## After /apply

Run `/verify` and `/review-pr` again after any applied automation to confirm the new branch state is still healthy.

## Disabling Dry-Run Permanently

To return to safe preview mode, restore `"simulateOnly": true` or `"automation.dryRunByDefault": true`.
